¿Porque?

¿Porque JavaScript?

El objetivo del Bootcamp es que aprendais a hacer el BackEnd y el FrontEnd de las aplicaciones, y con un solo lenguaje vamos a poder hacerlo.

JavaScript nace para darle funcionalidad e interactividad a las paginas web.

Google tiene un motor que ejecuta el JavaScript en Chrome que se llama V8 que es el motor de JavaScript mas rapido del mundo y lo usan otros navegadores como Opera, Safari, Edge, etc.

Google liberó el código de V8 en 2008 y desde entonces se ha convertido en el motor de JavaScript mas usado

A raiz de esto, Node JS nace para poder ejecutar JavaScript en el lado del servidor.

JavaScript es el lenguaje mas usado en el mundo.

Las bases de programación que vais a aprender en este Bootcamp como variables, funciones, bucles, condicionales, arrays, objetos, clases, ... son las mismas que se usan en otros lenguajes de programación como Java, C, C++, Python, etc. Lo que cambia es la sintaxis.

Navegador

Navegador

  • Motor JavaScript

    El código es revisado por un analizador que lee nuestro código línea a línea y revisa que la sintaxis es correcta.

    Este analizador conoce todas las reglas de JavaScript y puede detectar posibles errores. Si este fuera el caso nos devolvería un error y la ejecución se detendría.

    Si todo es correcto el analizador creará una estructura de datos conocida como Árbol de Sintaxis Abstracta (Abstract Syntax Tree), que luego se traduce al código de la máquina.

    Este nuevo código ya no es código JavaScript, sino una serie de instrucciones que pueden ser ejecutadas directamente por el procesador de nuestro ordenador. Es en este momento cuando nuestro código se ejecuta y se pone en funcionamiento.

  • Contexto de Ejecución

    El contexto de ejecución es el entorno donde se ejecuta el código JavaScript. Este contexto puede ser el navegador, NodeJS, etc.

    Ese entorno es llamado contexto de ejecución (execution context). Caja que almacena variables y funciones, aquí se evalua y ejecuta el código.

    El contexto de ejecución por defecto es el contexto global. Aquí se ejecuta todo el código que no se encuentre dentro de una función.

    Cuando llamamos a una función se crea un nuevo contexto de ejecución que es situado encima del contexto actual formando la llamada pila de ejecución o execution stack.

    Cada vez que un nuevo contexto de ejecución se crea, este se agrega a la parte superior de la pila de ejecución.

    El navegador siempre ejecutará el contexto de ejecución actual, el que esté en la cima de la pila de ejecución.

    El contexto de ejecución se rige por el método LIFO(Last In First Out) , que significa “último en entrar, primero en salir”

    Cuando el contexto de ejecución actual termina, se elimina de la pila de ejecución y el navegador ejecuta el contexto de ejecución que se encuentra debajo de él.

    Ejemplo

    'use strict'; const myVariable = 'Yo estoy declarada en el contexto global.'; console.log('Contexto Global: estamos en el contexto global'); function first() { console.log('Contexto Local 1: estamos dentro "first()"'); second(); console.log('Contexto Local 1: otra vez dentro de "first()"'); } function second() { console.log('Contexto Local 2: dentro de "second()"'); third(); console.log('Contexto Local 2: otra vez dentro de "second()"'); } function third() { console.log('Contexto Local 3: dentro de "third()"'); } first(); console.log('Contexto Global: regresamos al contexto global');

    El resultado de la ejecución del código anterior es el siguiente:

    Contexto Global: estamos en el contexto global Contexto Local 1: estamos dentro "first()" Contexto Local 2: dentro de "second()" Contexto Local 3: dentro de "third()" Contexto Local 2: otra vez dentro de "second()" Contexto Local 1: otra vez dentro de "first()" Contexto Global: regresamos al contexto global
Características

Características

JavaScript principalmente es un lenguaje del lado del cliente, se descarga en sus dispositivos y se procesa allí, pero también se usa en el lado del servidor como vais a aprender en el modulo de Node JS

JavaScript es un lenguaje de programación interpretado, es decir, no necesita ser compilado para ejecutarse. Se ejecuta directamente en el navegador.

Un Lenguaje interpretado es aquel que se ejecuta directamente sin necesidad de ser compilado.

Un Lenguaje compilado es aquel que se ejecuta a través de un programa que traduce el código fuente a código máquina (binario) para que pueda ser ejecutado por la máquina.

JavaScript es un lenguaje de programación orientado a objetos, es decir, que permite la creación de objetos y la manipulación de estos.

JavaScript es un lenguaje de programación de tipado devil y dinámico, es decir, que no es necesario declarar el tipo de dato de las variables. El tipo de dato se infiere en tiempo de ejecución. En otros lenguajes como Java, C o C++ es necesario declarar el tipo de dato de las variables cuando se crean.

Elementos JS

Principales elementos de JavaScript

  1. Variables:

    Tambien se les puede llamar identificadores. Contenedores para valores que se pueden cambiar a lo largo del tiempo (mutables) o no (inmutables). Se declaran con la palabra clave var, let o const.

    var está obsoleto y no se debe usar porque no es seguro, las variables declaradas con var pueden ser redeclaradas y reasignadas.

    let es la forma correcta de declarar variables en JavaScript. Las variables declaradas con let pueden ser reasignadas pero no redeclaradas.

    const es la forma correcta de declarar variables inmutables en JavaScript. Las variables declaradas con const no pueden ser reasignadas ni redeclaradas.

  2. Valores

    Los valores son los datos que se almacenan en las variables. Los valores pueden ser de tipo primitivo o de tipo objeto.

    Los valores de tipo primitivo son los siguientes: number, string, boolean, null y undefined.

    Los valores de tipo objeto son los siguientes: object, array, function, date, regexp, error, etc.

    Los valores de tipo primitivo son inmutables, es decir, no se pueden cambiar una vez creados ya que se almacenan en la pila de memoria.

    Los valores de tipo objeto son mutables, es decir, se pueden cambiar una vez creados ya que se almacenan en la pila de memoria.

  3. Operadores

    Los operadores son símbolos que se utilizan para realizar operaciones matemáticas, lógicas, de asignación, etc.

    Operadores aritméticos: +, -, *, /, %, **,++, --

    Operadores de asignación: =, +=, -=, *=, /=, %=, **=

    Operadores de comparación: ==, !=,===, !==, >, >=, <, <=

    Operadores lógicos: &&, ||, !

    Operadores de cadena: +, +=

    Operadores de condición ternaria: ?, :

  4. Expresiones

    Las expresiones son combinaciones de valores, variables y operadores que se evalúan a un valor.

  5. Funciones

    Las funciones son bloques de código que se pueden ejecutar varias veces. Las funciones se declaran con la palabra clave function y se invocan con los paréntesis ().

    Utilizan parametros (si los recibe) y mediante un conjunto de expresiones nos devuelve un valor a traves de la palabra reservada return

  6. Condicionales

    Los condicionales son estructuras de control que nos permiten ejecutar un bloque de código si se cumple una condición.

    Los condicionales más utilizados son los siguientes: if, else, else if, switch, case, default, while, do while, for, for in, for of, etc.

  7. Objetos

    Los objetos son una colección de propiedades, y una propiedad es una asociación entre un nombre (o clave) y un valor. Un valor de propiedad puede ser una función, que se denomina método del objeto. En JavaScript, casi "todo" es un objeto.

  8. Arrays

    Los arrays son una estructura de datos que nos permite almacenar varios valores en una sola variable. Los arrays se declaran con corchetes [] y se accede a sus elementos mediante su posición, que empieza en 0.

  9. Eventos

    Los eventos son acciones que se producen en el navegador y que podemos capturar con JavaScript. Los eventos más utilizados son los siguientes: click, mouseover, mouseout, keydown, keyup, keypress, load, unload, resize, scroll, change, submit, focus, blur, etc.

  10. DOM

    El DOM (Document Object Model) es una interfaz de programación para documentos HTML y XML. Es una representación estructural del documento, permitiendo modificar su contenido y presentación visual mediante código JavaScript.

  11. Clases

    Las clases son plantillas de objetos. Las clases se declaran con la palabra clave class y se instancian con la palabra clave new.

  12. Modulos

    Los módulos son bloques de código que se pueden importar y exportar de otros módulos. Los módulos se declaran con la palabra clave export y se importan con la palabra clave import.

  13. Fetch

    El método fetch() nos permite realizar peticiones HTTP y obtener la respuesta del servidor.

Use Strict

Use Strict

El modo estricto es una nueva característica de ECMAScript 5 que permite a los desarrolladores escribir código "seguro".

Cambia la sintaxis de JavaScript para evitar errores comunes.

Evita que se utilicen palabras reservadas como nombres de variables.

Evita que se utilicen variables no declaradas.

Elimina algunas funciones JavaScript que se consideran inseguras.

Evita que se utilicen argumentos duplicados en funciones.

Y más...

Basicamente, el modo estricto hace que nos salten errores que antes no salían.

Para activar el modo estricto, se debe escribir en la primera línea del script 'use strict';

Variables

Variables

¿Porque?

¿Porque usar Variables?

  • Se usan para almacenar información que se puede utilizar varias veces en el código.

  • Si tengo un valor y lo uso directamenten en el código, si quiero cambiarlo tengo que buscarlo en todo el código y cambiarlo uno a uno.

  • Si lo guardo en una variable, solo tengo que cambiar el valor de la variable y se cambia en todo el código.

Nomenclatura

Nomenclatura de variables

  • Deben tener un nombre descriptivo.
  • No pueden empezar con números.
  • Deben empezar con una letra, guion bajo (_) o signo de dólar ($).
  • Los siguientes caracteres después del primer carácter pueden ser números(0-9), letras(a-z, A-Z), guion bajo (_), signo de dólar ($) o la mayoría de las letras ISO 8859-1 o Unicode.
  • No pueden contener espacios.
  • No pueden contener caracteres especiales.
  • No pueden contener palabras reservadas.
  • Se recomienda utilizar camelCase.
  • Seran declaradas con const si no se van a reasignar.
  • Ser declararan con let si se van a reasignar.
Declaración

Declarar una variable

  • Se utilizan las palabra reservada var, let o const.
  • var: Esta sintaxis está obsoleta, por tanto, no debemos utilizarla bajo ningún concepto. Sigue funcionando y encontrarás muchos ejemplos de código JavaScript usandola. Pero no la uses 😉.
  • let y const fueron incorporadas en la versión ECMAScript 6 del 2015
  • let: Esta palabra reservada puede ser usada para declarar una variable que puede sufrir cambios en un futuro. let edad
  • const: Sirve para declarar variables que no sufrirán alteración ninguna en ningún momento posterior. Es obligatorio inicializarlas en el momento de su declaración. Si intentamos cambiar el valor de una variable declarada con const, nos saltaría un error advirtiendo que no podemos hacerlo. const nombre='Pedro'
  • Se recomienda utilizar const por defecto.
  • Es buena práctica declarar las variables al principio del script.
  • Se pueden declarar varias variables en una misma línea separándolas con comas.
Inicialización

Inicializar una variable

  • Se utiliza el operador =.
  • Se le asigna un valor a la variable.
  • Decarar: let myVariable
  • Inicializar: myVariable = 0
  • Se recomienda que si no se sabe el valor que va a tener una variable, se inicialice con el valor null.
Declarar e inicializar

Declarar e inicializar una variable

  • Es hacer las dos cosas a la vez.
  • let myvariable = 0
Reasignar

Reasignar una variable

Se utiliza el operador =.

Se le asigna un nuevo valor a la variable.

let myVariable = 0 myVariable = 1
Valores

Valores, Tipos de datos

Primitivos

Primitivos

  • Number

    Se usa para representar números enteros (integers) y números de punto flotante (float)

    También para valores numéricos especiales: Infinity, -Infinity y NaN (not a number)

    El 0 es un número con signo, por lo que hay dos ceros: el positivo 0 y el negativo -0 pero tienen el mismo valor.

  • BigInt

    Se usa para representar números enteros de cualquier tamaño.

    Se escriben añadiendo n al final del número.

    let myNumber = 1234567890123456789012345678901234567890n

  • String

    Se usa para representar datos de texto.

    Un string puede estar vacío o contener cualquier número de caracteres.

    Los caracteres pueden ser letras, dígitos, espacios, signos de puntuación, etc.

    Los strings se escriben entre comillas simples, dobles o template strings.

  • Boolean

    Se usa para representar un valor lógico que solo puede ser true(1) o false(0).

  • Null

    Se usa para representar un valor nulo.

    Es un valor especial que no es igual a nada pero al operar con él se convierte en 0.

    Se emplea cuando se quiere asignar un valor a una variable, pero no se sabe qué valor asignarle.

  • Undefined

    Se usa para representar un valor no definido.

    Si una variable no tiene ningún valor asignado, su valor es undefined.

    No se recomienda dejar una variable sin valor, ya que puede dar lugar a errores, tampoco asignarle undefined.

Objetos

Objetos

  • Object

    Se usa para representar datos estructurados.

    Un objeto es una colección de propiedades, y una propiedad es una asociación entre un nombre (o clave) y un valor.

    Los objetos se escriben entre llaves {}.

    Las propiedades se separan por comas.

    Las propiedades se escriben como un nombre(clave) seguido de dos puntos y un valor.

    Los valores pueden ser de cualquier tipo, incluso otros objetos.

    Si el valor es un string, se escriben entre comillas.

    Si el nombre de una propiedad es una palabra reservada, se debe escribir entre comillas.

    Si el nombre de una propiedad y el valor son estrictamente iguales, se puede omitir el valor.

  • Array

    Se usa para representar una lista ordenada de valores.

    Un array es una colección de valores.

    Los arrays se escriben entre corchetes [].

    Los elementos de un array pueden ser de cualquier tipo, incluso otros arrays.

    Los elementos de un array se separan por comas.

    Los elementos de un array se numeran, comenzando en 0.

  • function

    Se usa para representar una función.

    Las funciones se declaran con la palabra clave function.

  • Date

    Se usa para representar una fecha y hora.

    Las fechas se declaran con la palabra clave new seguida de Date.

    Un objeto Date se puede asignar a variables, objetos, arrays, etc.

    Un objeto Date se puede pasar como argumento a una función.

    Un objeto Date se puede devolver como valor de una función.

console.log()

console.log()

console es un objeto que nos permite interactuar con la consola del navegador.

log() es un método del objeto console que nos permite mostrar información en la consola del navegador.

Es muy útil para depurar código (debbugear) que consiste en ir comprobando en cada paso que se está haciendo lo que se quiere.

Para mostrar información en la consola, se debe escribir console.log() y entre paréntesis lo que se quiere mostrar.

Se puede mostrar texto, números, variables, arrays, objetos, etc.

Se puede mostrar más de un elemento separándolos con comas.

Hasta que aprendais más herramientas vamos a utilizar la consola constantemente para mostrar información.

prompt()

prompt()

prompt() es un método que nos permite mostrar un cuadro de diálogo en el navegador para que el usuario introduzca información.

Se debe escribir prompt() y entre paréntesis el texto que se quiere mostrar.

El método prompt() devuelve el valor que el usuario ha introducido en el cuadro de diálogo.

Se puede guardar en una variable para utilizarlo después.

Números

Math

Objeto Math

  • Math.PI

    Math.PI devuelve el valor de PI.

    console.log(Math.PI); //3.141592653589793

  • Math.abs()

    Math.abs() devuelve el valor absoluto.

    console.log(Math.abs(-3)); //3 console.log(Math.abs(3)); //3

  • Math.ceil

    Math.ceil() devuelve el entero aproximando hacia arriba.

    console.log(Math.ceil(3.5)); //4 console.log(Math.ceil(3.4)); //4

  • Math.floor

    Math.floor() devuelve el entero aproximando hacia abajo.

    console.log(Math.floor(3.5)); //3 console.log(Math.floor(3.4)); //3

  • Math.round

    Math.round() devuelve el valor de x redondeado al entero más cercano (0.5 se redondea al entero superior, 0.49 se redondea al entero inferior)

    console.log(Math.round(3.5)); //4 console.log(Math.round(3.4)); //3

  • Math.max

    Math.max() devuelve el número más alto de la lista de números indicados.

    console.log(Math.max(3, 5, 1, 2, 4)); //5

  • Math.min

    Math.min() devuelve el número más bajo de la lista de números indicados.

    console.log(Math.min(3, 5, 1, 2, 4)); //1

  • Math.pow

    Math.pow() devuelve la base elevada al exponente.

    console.log(Math.pow(2, 3)); //8

  • Math.sqrt

    Math.sqrt() devuelve la raíz cuadrada del número indicado.

    console.log(Math.sqrt(9)); //3

  • Math.random

    Math.random() devuelve un número aleatorio entre 0 y 1.

    console.log(Math.random()); //0.123456789
Number

Objeto Number

  • Number.isInteger

    Number.isInteger() devuelve true si el valor indicado es un número entero.

    console.log(Number.isInteger(3)); //true console.log(Number.isInteger(3.5)); //false console.log(Number.isInteger('Hola')); //false

  • Number.isFinite

    Number.isFinite() devuelve true si el valor indicado es un número finito.

    console.log(Number.isFinite(3)); //true console.log(Number.isFinite(3.5)); //true console.log(Number.isFinite('Hola')); //false console.log(Number.isFinite(Infinity)); //false

  • Number.isNaN

    Number.isNaN() devuelve true si el valor indicado es NaN.

    console.log(Number.isNaN(3)); //false console.log(Number.isNaN('Hola')); //false console.log(Number.isNaN(NaN)); //true
Métodos

Métodos

  • parseInt

    parseInt() devuelve un número entero a partir de un string.

    Si el string no empieza por un número, devuelve NaN.

    let a = '12';
    console.log(parseInt(a)); //12 console.log(parseInt('Hola')); //NaN

  • parseFloat

    parseFloat() devuelve un número decimal a partir de un string.

    Si el string no empieza por un número, devuelve NaN.

    let a = '12.5';
    console.log(parseFloat(a)); //12.5 console.log(parseFloat('Hola')); //NaN
